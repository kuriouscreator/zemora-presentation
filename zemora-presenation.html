Step 04 — Code

Claude Code reads the same Figma file Kevin was already building from.

Kevin had faithfully matched Tiger's designs in code — colors, spacing, layout all precise. The gap: no Figma MCP, so the token system in Figma and the values in Kevin's code weren't the same source of truth. MCP closes that loop permanently.

What Figma MCP enables
Claude queries the live Figma file — reads components, styles, layer names, variable collections. Token names in Figma become the same token names in Kevin's code. No translation layer, no drift.
What changes for Kevin
Kevin's implementation stays. The token system wraps around it. When a design variable changes in Figma, it propagates to Kevin's code via the connected pipeline — no manual hunting.

Ongoing

Figma and code stay in sync. New tokens get caught and added as the product grows.

This isn't a one-time handoff. The system stays connected — but it does need tending. When new components or values are added on either side, a quick Claude sync catches what's missing and adds it in.

01
Edit variables in Figma
Change a token. Every component using it responds instantly across the whole file.
02
Claude syncs both sides
"Read Figma variables and code tokens. Find anything missing on either side. Add it." Quick process — not automatic, but fast.
03
Monthly audit — ~4 minutes
Claude checks Figma variables vs code. Flags drift and missing tokens. Fixes them. System stays clean and current.
